​ Week 9 is an interesting week. We still focused on transport layer with its incredible protocols. The first question is about UDP, another communication protocol that we use in TL. As we said in week 2, in the TCP/IP model, users could implement both connection-oriented and connectionless communications in the Transport layer. TCP achieves connection-oriented communication. And UDP is the reason for connectionless communication. Internet Protocol in the internet layer of the TCP/IP model could implement connectionless communication for us. That is why we said, in this model, we could use only connectionless communication in the Internet layer. If you understand what happens in TCP/IP model, it is easy to remember corresponding characteristics of the OSI model. OSI model could only implements __________ communication in Network Layer and ________ communication in Transport Layer. I will leave blank for you to fill in. You could find answers in week2 slides.

​ Then we discussed TCP Tahoe and TCP Reno. The main difference between them is that Reno avoids Slow start when the window size hits the threshold that we set.

​ Following content is about the Leaky Bucket algorithm. This algorithm could help us to control sending speed and receiving speed. In other words, it can avoid congestions in the network by setting up a bucket at the receiver end.

​ The last content is about the pros and cons of timers.
