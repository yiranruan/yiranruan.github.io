​ On week 11, we have talked about several application systems and protocols. The first important system is DNS (Domain Name System). The problem why we need this system is because for human beings, it is rarely for us to remember the IP address of one sepefic server. We need a system for resolving URL link. Different zones are divided and allocated to one or several networks. Some well-known zones, for example, are “org” for orginazations, “edu” for educations, “au” for Australian websites. When we want to do IP address resolution. We send the URL link (for example, www.cis.unimelb.edu.au) to local name server. If this name server know the answer, it will return us the IP address. What we get from this local name server is exactly the final answer that we need. This mode is called recursive mode. If this name server does not know the IP address, it will send the same query to root name server. The root name server notice the “au” zone that we use. This server would return the IP address for name server which belongs to “au” zone. Then the local name server would send another query to “au” name server. That server would return IP address of “edu” name server. For each query that the local name server made, this server would get a partial answer and use that IP address to find next name server untill it get the IP address for URL link. We call this process as iterative mode.

​ Then we discussed about how to use cache and cookie to improve our experience in Network. The cookie could help server to recognise the user would want to access it again and provide a better service based on history recording that it saved in its database.

​ Cache is a internal or external space that we used to store web page when we download it from servers. It could help us access this web quickly and reduce the load on server end. The issue for cache is that we need to figure out whether the server administrators has updated thier web pages. It is hard to do predictions. The first strategy to solve this problem is to allocate a expire date for each web page that we download. Another strategy is to send a Conditional GET signal to the other end and ask the server whether they have updated their web page. If did, the server would send reply including the newest web page. If didn’t, the server would send another signal to indicate the cache that it could delivery the stored page to the user.
