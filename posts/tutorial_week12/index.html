The topic of this week is about security. We have discussed about four areas/aspects of Security. The first question is refered to secrecy. The process of secrecy could be represented by the formula as follows.

$$ D_{K2}(E{K_1}(P)) = P $$

If we set $K_1 = K_2$, this algorithm is symmetric algorithm. And if we set $K_1 \neq K_2$, we call this algorithm as asymmetric algorithm.

RSA is one kinds of asymmetric algorithms, which use two kinds of keys, public key and private key to do encryption and decryption. The steps and examples of how to generate those keys and how to do encryption and decryption are specified in my slides.

Next toptic is about how to let Alice cannot refute the fact that a document was signed by herself. We call it non-repudiation. To achieve this goal, we could use the property of RSA. When Alice sent the document to Bob, she has to sign her document by using her private key to encrypt her document. This cipertext would be sent to Bob attached on the document. When Bob received this document, he could do decryption by using the public key of Alice. If it works, it means this document is encrypted by using the private key of Alice. And we know the fact that only Alice has the private key of Alice. Hence, this document could only be signed and sent by Alice. Not anyone else.

We have known how asymmetric algorithms work in network. How about symmetric algorithm? If we set $K_1 = K_2$, it seems like we cannot send the key to other end directly, which would reveal our key to the public. Hence, we need another system to do key exchange, which is the topic of Question3 and Question4.

Then we talked about Integrity control in question 3. Even though it seems like it is impossible for a third guy to break Diffie-Hellman key exchange system, due to large primes n and g. The third guy, Trude, could generate her number z to create session keys with Alice and Bob respectively. When the communication starts, Alice would think the guy who communicates with her is Bob and Bob would think that guy is Alice. The message from Alice and Bob would be known and tempered with by Trudy.

The reason why former key exchange system is not safe is because for both Alice and Bob, it is hard for them to make sure that the guy who communicates with them is the guy they want to talk with. And this is another topic in Network security, authentication. To reach this goal, at the beginning, Alice asks a server to send her the public key of Bob and generate a identification $R_A$. Alice uses the key to encrypt a message which include the name of herself and the identification that she generated. This message is received by Bob. Bob opens the message by using his private key. Bob asks the same server to fetch the public key of Alice. Then Bob generate his identification $R_B$ and the session key that he want to use with Alice. The name of Bob, $R_A$, $R_B$ and session key would be put in one message and encrypted by using the public key of Alice. The message is received by Alice. When Alice uses her key to open this message, she would see her identification. As for now, we know the fact that the message which was encrypted by Bob’s public key could be only openned by Bob’s private key. Hence, only the owner of Bob’s private key could read the message and get $R_A$. And another fact shows that only Bob has his private key. Hence, the guy who sent this message which include $R_A$ is able to be and can only be Bob himself. So we let Alice to be comfirm the identification of Bob. Then Alice uses the session key she received to encrypt the $R_B$ and sent it to Bob. For same reason as before, Bob could identify the identification of Alice.
