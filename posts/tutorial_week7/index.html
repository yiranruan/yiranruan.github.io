​ In this week, we have discussed about MACA protocol, IP address (mainly about prefix and subnetmask) and fragmentation.

​ MACA, which is an wonderful protocol to help us aviod collisions withour carrier sense medium. The basic idea is that for the sender to stimulate the receiver into outputting a short frame, so stations nearby can detect this transmission and avoid transmitting for the duration of the upcoming (large) data frame. RTS (Request to send) and CTS (Clear to send) are two short frame (about 30 bytes) which work as indicators to indicate every stations in the range of A and B to keep slience while they are transmitting. The size of data would be included in RTS and CTS to help other stations to calculate how long that they need to be quiet.

​ Prefix and Subnetmask are used for recognize and extract network portion from IP address. Prefixes are written by giving the lowest IP address in the block and the size of the block. The length of the prefix corresponds to a binary mask of 1s in the network portion for Subnet mask.

​ Fragmentation is last content that we discussed. An obvious internetworking problem appears when a large packet wants to travel through a network whose maximum packet size is too small. We introduced fragmentation to solve this problem. There are two kinds of fragmentations, transparent fragmentation and non-transparent fragmentation.
